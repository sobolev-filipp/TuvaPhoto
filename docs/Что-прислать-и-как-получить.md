# Что прислать Claude и как это получить

Памятка для владельца: какие данные, ключи и картинки нужны, чтобы включить
«живые» функции (реальные платежи, письма, вход через соцсети) и наполнить сайт
фотографиями. Здесь же — форматы картинок и промпт для картинки на странице входа.

> **Коротко по порядку действий:** заводите нужный сервис → берёте оттуда ключи →
> кладёте их в файл `apps/api/.env` (см. ниже) → пишете мне «включил ЮKassa» →
> я подключаю код. Картинки просто присылаете мне файлами.

---

## 0. Как безопасно передавать ключи и пароли

- **Секреты (ключи, пароли, токены) НЕ присылайте в переписку и НЕ коммитьте в git.**
  Claude по правилам безопасности не вводит и не хранит чужие пароли/ключи.
- Все секреты живут только в файле **`apps/api/.env`** на сервере (он в git не
  попадает — в `.gitignore`). Открываете файл, дописываете строку вида `КЛЮЧ=значение`,
  сохраняете, перезапускаете API.
- Мне достаточно написать: «завёл ЮKassa, ключи положил в .env» — я допишу код,
  который эти переменные читает. Сами значения мне видеть не нужно.
- Если ключ «засветился» (случайно отправили) — отзовите его в личном кабинете
  сервиса и выпустите новый.

## 1. Онлайн-платежи (реальная оплата вместо текущей заглушки)

Сейчас оплата — «мок»: страница `/pay/:token` просто отмечает сумму, реальные
деньги не списываются. Чтобы принимать деньги, нужен платёжный провайдер.

### Рекомендую: ЮKassa (бывшая Яндекс.Касса)
Поддерживает карты и **СБП**, работает с самозанятыми и ИП, популярна в РФ.

**Что сделать:**
1. Зарегистрироваться на **yookassa.ru**, подключить магазин (нужны ИНН/данные
   самозанятого или ИП, реквизиты для вывода).
2. В личном кабинете: **Настройки → Магазин → Данные для API**.
3. Взять два значения: **shopId** (идентификатор магазина) и **секретный ключ**
   (Secret Key, начинается на `live_` для боевого режима или `test_` для теста).
4. Там же указать **URL для уведомлений (webhook)**: `https://ВАШ-ДОМЕН/api/payments/yookassa/webhook`
   (этот адрес я сделаю при подключении).

**Что положить в `apps/api/.env`:**
```
YOOKASSA_SHOP_ID=ваш_shopId
YOOKASSA_SECRET_KEY=ваш_секретный_ключ
```
**Что дальше делаю я:** подключаю SDK ЮKassa, создаю платёж при оплате/доплате,
принимаю webhook об успешной оплате и автоматически обновляю статус заказа.

Начать можно с **тестового режима** (ключи `test_...`) — деньги не списываются,
но весь путь оплаты проверяется по-настоящему.

### Альтернатива 1: Т-Касса (Тинькофф Эквайринг)

Подходит для ИП/ООО и самозанятых, поддерживает карты и СБП.

**Что сделать:**
1. Оставить заявку на **tbank.ru/business/internet-acquiring** (или в приложении
   Т-Бизнес). Подключение интернет-эквайринга, подписание договора.
2. После подключения в личном кабинете **Т-Бизнес → Магазины → ваш магазин →
   Терминалы** появится терминал. Взять:
   - **Terminal Key** (идентификатор терминала),
   - **Пароль терминала** (Password / секретный ключ для подписи запросов).
3. Там же указать **адрес уведомлений (Notification URL / webhook)**:
   `https://ВАШ-ДОМЕН/api/payments/tinkoff/webhook`
4. Обычно дают отдельный **тестовый терминал** — сначала проверяем на нём.

**Что положить в `apps/api/.env`:**
```
TINKOFF_TERMINAL_KEY=ваш_terminal_key
TINKOFF_PASSWORD=пароль_терминала
```
**Что делаю я:** подключаю приём платежей по API Т-Кассы (метод `Init` →
редирект на оплату → `Notification` webhook об оплате → обновляю статус заказа).
Подпись запросов (token) считается из пароля терминала.

### Альтернатива 2: Сбербанк Эквайринг (SberPay / интернет-эквайринг)

Через **Сбербанк** (платёжный шлюз, бывш. «Сбербанк Эквайринг» на базе RBS).

**Что сделать:**
1. Подать заявку на интернет-эквайринг в **СберБизнес** (sberbank.ru/business →
   Эквайринг → Интернет-эквайринг). Нужен расчётный счёт в Сбере, договор.
2. После подключения выдают доступ к **платёжному шлюзу**. Взять:
   - **Логин мерчанта (userName)** — вида `имя-api`,
   - **Пароль (password)** для этого логина,
   - (иногда вместо логина/пароля дают **token** — одноразовый ключ доступа к API;
     подойдёт и он).
3. Указать адрес **уведомлений/возврата** (Callback/Return URL):
   `https://ВАШ-ДОМЕН/api/payments/sber/webhook`
4. Есть **тестовый шлюз** (`3dsec.sberbank.ru`) с тестовыми логином/паролем —
   сначала проверяем на нём, потом переключаем на боевой (`securepayments.sberbank.ru`).

**Что положить в `apps/api/.env`:**
```
SBER_USERNAME=ваш_логин-api
SBER_PASSWORD=ваш_пароль
# либо, если выдали токен вместо логина/пароля:
# SBER_TOKEN=ваш_token
SBER_GATEWAY=test        # test — тестовый шлюз, live — боевой
```
**Что делаю я:** подключаю REST-шлюз Сбера (`register.do` → редирект на оплату →
проверка статуса `getOrderStatusExtended` / callback → обновляю статус заказа).

> **Как выбрать:** если уже есть счёт в Сбере — проще Сбербанк Эквайринг; если в
> Тинькофф/Т-Бизнес — Т-Касса; для самозанятого без ИП чаще всего быстрее всего
> заходит **ЮKassa**. Технически мне подходит любой — скажите, что подключили, и
> положите ключи в `.env`.

## 2. Отправка писем (SMTP) — коды входа и уведомления

Сейчас коды двухфакторной проверки и письма печатаются в консоль сервера (для
разработки). Чтобы они уходили на почту клиентам, нужен SMTP-доступ.

**Варианты, где взять ящик для рассылки:**
- **Яндекс 360 для бизнеса** (почта на вашем домене) — надёжно для РФ.
- **Mail.ru для бизнеса**, **Selectel**, любой хостинг с почтой.
- Личный ящик Яндекс/Mail.ru — тоже можно, но нужен **пароль приложения**
  (не основной пароль!), создаётся в настройках безопасности ящика.

**Что понадобится:**
- SMTP-хост (например, `smtp.yandex.ru`),
- порт (`465` для SSL или `587` для STARTTLS),
- логин (полный email) и **пароль приложения**,
- адрес отправителя (от кого письма).

**Что положить в `apps/api/.env`:**
```
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=465
SMTP_USER=studio@ваш-домен.ru
SMTP_PASS=пароль_приложения
SMTP_FROM=ТуваФото <studio@ваш-домен.ru>
```
**Что дальше делаю я:** переключаю отправку на реальный SMTP (код уже готов к этому).

## 3. Вход через соцсети (Яндекс ID, VK ID) — по желанию

Обычный вход по email+паролю уже работает. Кнопки «Войти через Яндекс/VK»
требуют регистрации приложения.

### Яндекс ID
1. **oauth.yandex.ru** → «Создать приложение».
2. Разрешения: доступ к email и имени пользователя.
3. **Redirect URI:** `https://ВАШ-ДОМЕН/api/auth/oauth/yandex/callback`
4. Взять **Client ID** и **Client Secret**.

### VK ID
1. **id.vk.com / dev.vk.com** → создать приложение.
2. **Redirect URI:** `https://ВАШ-ДОМЕН/api/auth/oauth/vk/callback`
3. Взять **ID приложения** и **Защищённый ключ (Secure key)**.

**Что положить в `apps/api/.env`:**
```
YANDEX_CLIENT_ID=...
YANDEX_CLIENT_SECRET=...
VK_CLIENT_ID=...
VK_CLIENT_SECRET=...
```

## 4. Картинки для ГЛАВНОЙ страницы (карусель, где фото меняются)

На главной вверху — большой баннер (hero), где фоновые фото плавно **сменяют друг
друга**. Сейчас там заглушки. Пришлите мне 3–6 фотографий — я поставлю их в карусель.

**Требования к фото карусели:**
- **Ориентация:** горизонтальная (пейзажная).
- **Размер:** от **1920×1080** до **2400×1400** пикселей (чем больше, тем чётче на
  больших экранах; но не гигантские — до ~4000 px по ширине).
- **Формат:** **JPG** или **WebP** (WebP легче). PNG не нужен — фото тяжёлые.
- **Вес файла:** желательно до **~600 КБ** каждое (сожмём при необходимости).
- **Композиция:** важные детали держите **правее/по центру** — слева на фото ложится
  тёмный градиент и текст («Память, которую хочется листать»), там лицо/логотип
  перекроется.
- **Сюжеты:** атмосферные кадры выпускных, детей, съёмочного процесса, готовых
  альбомов в руках. 3–6 штук — карусель будет их перелистывать.

Пришлите файлами (можно архивом) — я добавлю их в карусель и подпишу.

## 5. Картинка для страницы ВХОДА (сгенерированная)

На странице входа слева — большая вертикальная картинка «под тематику фотографа».
Сейчас там заглушка. Сгенерируйте картинку в любом ИИ-генераторе (Kandinsky,
Midjourney, DALL·E, Шедеврум и т.п.) по промпту ниже и пришлите мне файл.

**Требования:**
- **Ориентация:** вертикальная (портретная), панель слева высокая и узкая.
- **Размер:** примерно **1200×1600** или **1400×1800** пикселей (соотношение 3:4).
- **Формат:** JPG или WebP.
- **Тон:** тёмный, премиальный, с тёплым/золотистым светом — под стиль сайта
  (чёрный фон + золото). Оставьте **низ и левый край** спокойными/затемнёнными —
  на них ложится градиент и текст «Ваши альбомы всегда под рукой».

**Промпт (рус.):**
> Атмосферная вертикальная фотография в тёмном премиальном стиле: фотограф в мягком
> тёплом свете держит открытый фотоальбом выпускников, вокруг лёгкое боке и золотистые
> блики, глубокие тени, кинематографичное освещение, элегантно и уютно. Настроение —
> тёплые воспоминания и качество. Пустое затемнённое пространство слева и снизу для
> текста. Соотношение сторон 3:4, высокое разрешение, без текста и логотипов.

**Prompt (eng., для Midjourney/DALL·E):**
> Atmospheric vertical photograph, dark premium mood: a photographer in soft warm light
> holding an open graduation photo album, gentle bokeh and golden highlights, deep
> shadows, cinematic lighting, elegant and cozy. Theme: warm memories and craftsmanship.
> Keep the left side and bottom dark and empty for text overlay. Aspect ratio 3:4, high
> resolution, no text, no logos. --ar 3:4

Если захотите другой сюжет (стопка альбомов, камера на столе, выпускники листают
книгу) — поменяйте первую фразу промпта, остальное оставьте.

## 6. Остальные фото (можно загружать самому в админке)

Эти картинки **не нужно присылать мне** — вы загружаете их сами в админ-панели,
формат один: **JPG, PNG или WebP, до 15 МБ**.

- **Фото фотографа** — раздел «О фотографе». Лучше квадрат или портрет, от 800×800.
- **Обложки** (передняя/задняя) — раздел «Обложки».
- **Развороты альбомов** — при создании альбома и демо-ссылок.
- **Демо для клиента** — раздел «Готовые для клиента».

Совет: снимайте/экспортируйте в **sRGB**, иначе цвета в браузере поплывут.

## 7. Домен и запуск (когда будете выходить «в прод»)

- Понадобится **домен** (например, `tuvafoto.ru`) и **российский сервер/VPS** (152-ФЗ —
  данные граждан РФ хранить в России). Подробности разворачивания — в `docs/DEPLOY.md`.
- После покупки домена дайте знать — пропишем его в настройках (адреса webhook,
  redirect URI для соцсетей и т.д. используют этот домен).

|---|---|---|
