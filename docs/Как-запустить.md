# Как запустить сайт и проверить его самому

Инструкция для Windows. Все команды выполняются в **PowerShell**.

---

## 1. Что нужно установить один раз

**Node.js версии 20 или новее.** Скачать: <https://nodejs.org> — берите кнопку **LTS**,
устанавливайте со всеми галочками по умолчанию.

**Docker Desktop** — в нём работает база данных. Скачать: <https://www.docker.com/products/docker-desktop>.

Проверить, что установилось (откройте PowerShell заново после установки):

```powershell
node -v
docker -v
```

Должны появиться номера версий. Если пишет «не является внутренней или внешней
командой» — перезагрузите компьютер и попробуйте снова.

> PostgreSQL отдельно ставить **не нужно** — он живёт внутри Docker.

---

## 2. Первый запуск — один раз

1. **Запустите Docker Desktop** (найдите в меню «Пуск» и откройте). Подождите,
   пока значок кита в трее перестанет мигать — значит, он готов.

2. Откройте PowerShell и перейдите в папку проекта:

   ```powershell
   cd "F:\Проекты\ТуваФото"
   ```

3. Создайте файл с настройками сервера (пароли, ключи). Команда ниже создаёт
   его, только **если его ещё нет** — уже настроенный `.env` не затрёт:

   ```powershell
   if (-not (Test-Path apps\api\.env)) { Copy-Item apps\api\.env.example apps\api\.env }
   ```

   > ⚠️ Не выполняйте `Copy-Item ... .env.example ... .env` напрямую, если `.env`
   > уже существует — это перезапишет ваши пароли и ключи значениями-заглушками.

4. Установите и подготовьте всё одной командой:

   ```powershell
   npm run setup
   ```

   Она поставит зависимости, поднимет базу данных и заведёт владельца сайта со
   справочниками. Занимает несколько минут, повторять не нужно.

---

## 3. Обычный запуск — каждый раз

1. **Запустите Docker Desktop** (если ещё не запущен) и дождитесь кита в трее.

2. В PowerShell из папки проекта:

   ```powershell
   cd "F:\Проекты\ТуваФото"
   npm start
   ```

   Эта одна команда поднимает **и базу, и оба сервера** — сайт и серверную часть.
   В окне побегут строки с пометками `[API]` (жёлтые) и `[WEB]` (голубые) — это
   два сервера работают вместе в одном окне.

3. Когда увидите строку вида `Local: http://localhost:5173/` — откройте в браузере
   **<http://localhost:5173>**.

**Окно PowerShell закрывать нельзя** — в нём работают серверы. Остановить всё —
`Ctrl + C` в этом окне. База при этом продолжит работать в Docker; чтобы
остановить и её:

```powershell
npm run db:stop
```

Данные не пропадут.

### Проверить, что всё поднялось правильно

Откройте <http://localhost:3000/api/health>. Должно ответить
`{"status":"ok","db":"up"}`.

- `db:down` или страница не открывается → **не запущен Docker Desktop**. Запустите
  его и снова `npm start`.
- Сайт открывается, но регистрация/вход выдаёт **502** → серверная часть не
  поднялась. Убедитесь, что запускали именно `npm start` (а не `npm run dev` —
  та поднимает только сайт без сервера), и что в окне есть жёлтые строки `[API]`.

> Хотите посмотреть только внешний вид сайта, без входа и регистрации? Тогда
> Docker и сервер не нужны — достаточно `npm run dev` (только сайт, демо-данные).

### Код подтверждения при входе

Настоящие письма пока не отправляются. Когда вы регистрируетесь или входите,
**код из 4 цифр печатается в том же окне PowerShell** (среди жёлтых строк `[API]`):

```
┌─────────────────────────────────────────────┐
│  КОД ПОДТВЕРЖДЕНИЯ (заглушка вместо письма) │
├─────────────────────────────────────────────┤
│  Кому:  Tuvafoto@mail.ru                    │
│  Код:   4878                                │
└─────────────────────────────────────────────┘
```

Возьмите код оттуда и введите на сайте. Когда подключим почту, он будет
приходить письмом, а из консоли исчезнет.

### Вход через Яндекс / VK

Кнопки соцвхода на формах появляются, только если заданы ключи приложения. Пока
их нет — блок «или войти через» просто не показывается, это нормально.

Чтобы включить, например, Яндекс:
1. Создайте приложение на <https://oauth.yandex.ru>.
2. В настройках Callback URI укажите
   `http://localhost:3000/api/auth/oauth/yandex/callback`.
3. Впишите выданные Client ID и Client Secret в `apps\api\.env`
   (`YANDEX_CLIENT_ID`, `YANDEX_CLIENT_SECRET`) и перезапустите сервер.

Для VK — то же самое на <https://vk.com/apps?act=manage>, строки `VK_CLIENT_ID`
и `VK_CLIENT_SECRET`, Callback `.../auth/oauth/vk/callback`.

### Вход владельца

Логин и пароль владельца лежат в файле `apps\api\.env` — строки `OWNER_EMAIL`
и `OWNER_PASSWORD`. Сейчас это `Tuvafoto@mail.ru` и `TuvaFoto-dev-2026`.
Пароль временный, для разработки: перед публикацией сайта его нужно сменить.

## 4. Что проверять

### Главная

- Фон вверху меняется сам каждые ~4 секунды; точки внизу переключают вручную.
- Кнопки **Заказать альбом** и **Позвонить фотографу**.
- Секция «Полистайте альбомы» — три книги.
- Отзывы, кнопка **Оставить отзыв** (открывает окно), плашка «Соберите свой альбом».
- Круглая золотая кнопка звонка в правом нижнем углу.

### Книга — главное, что стоит проверить внимательно

Книга есть на главной и на странице альбома.

- Клик по обложке — книга открывается.
- Стрелки **‹** и **›** — листают. Лист должен поворачиваться, а фото под ним
  меняться в середине поворота.
- **▶ Старт** — листает само по кругу, **⏸ Стоп** — останавливает.
- **⛶ Во весь экран** — книга разворачивается на весь экран. Закрыть: кнопка
  **✕ Закрыть**, клавиша `Esc` или клик по тёмному фону.
- В полноэкранном режиме листают ещё и стрелки на клавиатуре `←` `→`.
- Долистайте до конца — появится задняя обложка и надпись «Конец альбома».

### Каталог

- Кнопки категорий сверху фильтруют альбомы.
- Клик по обложке или названию — открывается страница альбома.
- Кнопка **В конструктор** (сам конструктор ещё в работе).

### Контакты

Данные фотографа, фото, кнопки соцсетей.

### Проверка на телефоне

Не обязательно доставать телефон — в браузере:

1. Нажмите `F12` (откроется панель разработчика).
2. Нажмите `Ctrl + Shift + M` — страница станет узкой, как на телефоне.
3. Сверху можно выбрать модель телефона.

Что смотреть:
- Вместо меню — кнопка **☰**, по нажатию выезжает панель справа.
- Кнопки **Заказать альбом** и **Позвонить фотографу** — во всю ширину экрана.
- Книга уменьшается и помещается в экран целиком, ничего не обрезано.

Вернуть обычный вид — снова `Ctrl + Shift + M`, закрыть панель — `F12`.

## 6. Проверка приложения (PWA)

Установку приложения и плашку обновления в режиме `npm run dev` проверить
нельзя — они работают только в «боевой» сборке. Для проверки:

```powershell
npm run build --workspace apps/web
npm run preview --workspace apps/web
```

Откройте адрес, который покажет `preview` (обычно <http://localhost:4173>).

**Установка приложения.** В адресной строке Chrome справа появится значок
установки. Либо меню `⋮` → «Установить приложение». Раздел с инструкцией —
<http://localhost:4173/install>.

**Плашка «Установите приложение»** появляется через пару секунд после захода и
потом не чаще одного раза в сутки. Чтобы увидеть её снова, не дожидаясь суток:
`F12` → вкладка **Application** → **Local Storage** → удалить строки, начинающиеся
на `tuvafoto:install`, и обновить страницу.

**Плашка «Доступно обновление»** появляется, когда вышла новая версия сайта.
Проверить можно так: не закрывая вкладку, соберите сайт заново (`npm run build`)
в другом окне PowerShell и обновите страницу — появится плашка с кнопкой
**Обновить**.

Остановить `preview` — `Ctrl + C`.

## 7. Если что-то пошло не так

**Регистрация или вход выдаёт 502**
Не поднялась серверная часть. Запускайте `npm start` (а не `npm run dev` — та
поднимает только сайт). Убедитесь, что Docker Desktop запущен, и что в окне бегут
жёлтые строки `[API]`.

**«npm не является внутренней или внешней командой»**
Node.js не установлен или не перезапущен PowerShell. Переустановите Node.js и
откройте новое окно PowerShell.

**«Port 5173 is already in use» или «Port 3000 is already in use»**
Серверы уже запущены в другом окне. Либо закройте то окно, либо остановите все
процессы и запустите заново:

```powershell
Get-Process -Name node | Stop-Process -Force
```

**Страница белая или сыпятся ошибки**
Нажмите `F12`, вкладка **Console** — пришлите мне текст ошибки.

**После обновления кода что-то сломалось**
Обычно помогает переустановка зависимостей:

```powershell
cd "F:\Проекты\ТуваФото"
Remove-Item -Recurse -Force node_modules
npm install
```

**Изменения не видны в браузере**
Обновите страницу с очисткой кэша: `Ctrl + Shift + R`.

**Сервер пишет `db:down`, или `docker` ругается на подключение**
Не запущен Docker Desktop. Откройте его из меню «Пуск», дождитесь, пока значок
кита перестанет мигать, и снова `npm start`.

**Не приходит код подтверждения**
Он и не придёт письмом — почта ещё не подключена. Код печатается в окне
PowerShell среди строк `[API]` (см. раздел «Код подтверждения» в пункте 3).

## 8. Сбросить данные в базе

Иногда нужно начать с чистого листа — стереть тестовых пользователей, заказы,
альбомы и вернуть заводское состояние (владелец с временным паролем, справочники).

**Docker Desktop должен быть запущен.** Из папки проекта:

```powershell
npm run db:reset
```

Эта команда **полностью очищает базу**, заново создаёт все таблицы и заполняет
их стартовыми данными (владелец + категории + виды съёмки + обложки).
Спросит подтверждение — введите `y` и Enter.

После сброса вход владельца снова: **логин `Tuvafoto@mail.ru`, пароль
`TuvaFoto-2026`** (временный, при первом входе сайт попросит сменить его и
заполнить данные о себе).

> ⚠️ Сброс удаляет **все** данные без возможности вернуть. Загруженные фото
> (когда появятся) он не трогает — только базу.

Полностью снести и базу, и её том Docker (совсем «с нуля»):

```powershell
docker compose down -v
npm run setup
```

## 9. Сохранить изменения в свой GitHub

Репозиторий привязан к **вашему** аккаунту:
`github.com/sobolev-filipp/-`. Коммиты подписываются вашим именем (`Filipp`).

Когда хотите сохранить сделанное и отправить к себе на GitHub — из папки проекта:

```powershell
git add -A
git commit -m "коротко, что изменилось"
git push
```

- `git add -A` — берёт все изменённые файлы.
- `git commit -m "..."` — сохраняет их одной точкой в истории (локально).
- `git push` — отправляет на GitHub.

Полезное:

| Что нужно | Команда |
|---|---|
| Посмотреть, что изменилось | `git status` |
| История коммитов | `git log --oneline -10` |
| Отменить незакоммиченные правки в файле | `git checkout -- путь\к\файлу` |

> Файл `apps\api\.env` с паролями в GitHub **не отправляется** — он в списке
> игнорируемых. Это правильно, так и должно быть.
