# Стек и решения

Фиксируем выбранное, чтобы не переобсуждать. Дизайн-эталон —
`Downloads/Веб-сайт для выпускных альбомов/design_handoff_tuvafoto` (в проде
прототипный фреймворк `support.js` / `*.dc.html` не используется).

## Frontend

- **React 19 + TypeScript + Vite**, роутинг — React Router.
- **Tailwind v4**, токены — в `apps/web/src/index.css` (`@theme`). Цвета,
  шрифты и радиусы правим только там.
- **Zustand** — состояние конструктора и авторизации, **TanStack Query** — данные с API.
- **3D-книга** — CSS-трансформы, без сторонних библиотек (`apps/web/src/components/Book.tsx`).
- **PWA** — `vite-plugin-pwa` в режиме `prompt`.

## Backend (следующий этап)

- **NestJS + Prisma + PostgreSQL.** Один язык с фронтом, общие DTO;
  WebSocket-гейтвей из коробки — нужен для realtime-заказов.
- **Realtime заказов:** Socket.IO, комната `admin`, событие `order.created`.
  Счётчик новых заказов хранится в БД (`Order.readAt`), а не только в сокете —
  иначе бейдж обнулялся бы при перезагрузке админки.
- **Файлы:** локальный диск VPS через абстракцию `StorageService` с тем же
  интерфейсом, что у S3 — переезд на Selectel/Yandex Object Storage сведётся
  к замене одной реализации. Ресайз — sharp.
- **Авторизация:** email + пароль с 2FA по коду на почту; Яндекс ID и VK ID —
  через OAuth (свой 2FA для них не нужен, провайдер уже подтвердил личность).
  MAX не подключаем. Владелец один, аккаунт заводится сидом (`role=OWNER`),
  регистрации админов нет.
- **Платежи:** абстракция `PaymentProvider` + мок. Реальный провайдер
  подключается, когда будут договор с банком и ключи.

## Инфраструктура

- Российский VPS (Timeweb/Selectel), Docker Compose: api + postgres + caddy.
  Персональные данные остаются в РФ (152-ФЗ).

## Правила, которые легко нарушить

- **Цену заказа бэкенд обязан пересчитать сам** (`domain/pricing.ts` — та же
  формула), сумма с клиента не является источником истины.
- Согласия 152-ФЗ и выбор cookie хранить с датой и версией текста политики.
- Публичные ссылки на готовые альбомы (`/share/:token`) — со сроком действия.
