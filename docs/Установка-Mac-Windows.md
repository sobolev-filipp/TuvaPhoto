# Установка и запуск проекта с нуля — macOS и Windows

Полная инструкция: как на чистом компьютере (Mac или Windows) поставить всё
необходимое, собрать проект и запустить его для теста. Проходите сверху вниз.

> Кратко после установки всё сводится к трём командам из папки проекта:
> `npm run setup` → `npm start` → открыть <http://localhost:5173>.
> Ниже — подробно, шаг за шагом.

---

## 0. Что вообще нужно

Для работы проекта на компьютере должны быть установлены:

| Инструмент | Зачем |
|---|---|
| **Node.js 20+** | среда, в которой работают сайт и сервер |
| **Docker Desktop** | в нём поднимается база данных PostgreSQL |
| **Git** | скачать код проекта (и сохранять изменения) |
| **PowerShell** (`pwsh`) | *опционально* — чтобы запускать команды в стиле Windows |

> PostgreSQL отдельно ставить **не нужно** — он живёт внутри Docker.

---

## 1. Установка инструментов

### macOS

1. **Homebrew** (менеджер пакетов). Откройте приложение «Терминал» и выполните:

   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```

   После установки следуйте подсказке в конце (добавить `brew` в PATH — команда
   вида `echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile`).

2. **Node.js, Git и Docker Desktop:**

   ```bash
   brew install node git
   brew install --cask docker
   ```

   > Если `--cask docker` выдаст ошибку (Homebrew иногда переименовывает cask),
   > поставьте Docker Desktop так: `brew install --cask docker-desktop`.

3. **PowerShell** (по желанию — чтобы команды `.env` совпадали с Windows):

   ```bash
   brew install powershell
   ```

   Проверить: `pwsh -v` покажет версию.

4. **Запустить Docker Desktop** (обязательно перед работой с базой):

   ```bash
   open -a Docker
   ```

   Первый запуск попросит подтвердить разрешения. Дождитесь, пока иконка кита
   в строке меню перестанет анимироваться (Docker Desktop → «Running»).

> **Apple Silicon (M1/M2/M3)** — всё работает без плясок: образ `postgres:17-alpine`
> собран под arm64.

### Windows

1. Установите три программы (проще всего через `winget` в PowerShell, либо
   скачайте инсталляторы с сайтов):

   ```powershell
   winget install OpenJS.NodeJS.LTS
   winget install Git.Git
   winget install Docker.DockerDesktop
   ```

   Сайты вручную: Node — <https://nodejs.org> (кнопка **LTS**),
   Git — <https://git-scm.com>, Docker — <https://www.docker.com/products/docker-desktop>.

   > Docker Desktop на Windows использует **WSL2** — при установке он всё настроит
   > сам; может потребоваться перезагрузка.

2. **PowerShell 7** (`pwsh`) уже есть на Windows как встроенный Windows PowerShell.
   Если хотите свежую версию:

   ```powershell
   winget install Microsoft.PowerShell
   ```

3. **Запустить Docker Desktop** — найдите его в меню «Пуск» и откройте. Дождитесь,
   пока иконка кита в трее перестанет мигать.

### Проверка (обе системы)

Откройте **новый** терминал и выполните:

```bash
node -v      # v20.x.x или новее
git --version
docker -v
docker compose version
```

Все команды должны показать номера версий. Если «команда не найдена» —
перезапустите терминал (а на Windows — компьютер) и повторите.

## 3. Настроить файл `.env` (секреты сервера)

Сервер читает настройки из `apps/api/.env`. Его нужно создать из шаблона
`.env.example`. Команда создаёт файл, **только если его ещё нет** — готовый `.env`
с вашими паролями не затрётся.

**macOS (bash / zsh):**

```bash
[ -f apps/api/.env ] || cp apps/api/.env.example apps/api/.env
```

**macOS через PowerShell** (обратите внимание — пути через `/`):

```bash
pwsh -Command "if (-not (Test-Path apps/api/.env)) { Copy-Item apps/api/.env.example apps/api/.env }"
```

**Windows (PowerShell)** — здесь пути через `\`:

```powershell
pwsh -Command "if (-not (Test-Path apps\api\.env)) { Copy-Item apps\api\.env.example apps\api\.env }"
```

> ⚠️ Не копируйте `.env.example → .env` напрямую, если `.env` уже существует —
> это перезапишет ваши пароли и ключи значениями-заглушками.

Значений по умолчанию достаточно для локального запуска. Соцвход (Яндекс/VK) и
реальная почта подключаются позже — см. комментарии внутри `.env.example`.

## 4. Первичная сборка — один раз

Docker Desktop должен быть **запущен** (кит готов). Из папки проекта:

```bash
npm run setup
```

Эта команда: ставит зависимости (`npm install`), **генерирует Prisma Client**,
поднимает базу в Docker (`docker compose up -d --wait`) и заполняет её стартовыми
данными — владелец сайта и справочники (категории, виды съёмки, обложки).
Занимает несколько минут.

> **⚠️ Новые версии Node/npm блокируют install-скрипты пакетов.** Если при
> `npm install` появились строки `npm warn allow-scripts … packages have install
> scripts not yet covered`, значит не собрались нативные зависимости (`argon2`,
> `esbuild`) и движки Prisma — сервер потом упадёт. Разрешите их **один раз** и
> переустановите:
>
> ```bash
> npm approve-scripts --allow-scripts-pending   # одобрить все ожидающие
> # если команда просит выбрать — одобрите: argon2, esbuild, fsevents, prisma, @prisma/engines
> npm install                                   # переустановить, чтобы скрипты отработали
> npm run setup                                 # повторить подготовку
> ```
>
> На привычных LTS-версиях Node (20/22) этого шага обычно не требуется.

## 5. Обычный запуск — каждый раз

1. Убедитесь, что **Docker Desktop запущен** (macOS: `open -a Docker`; Windows —
   из «Пуска»). Дождитесь кита.

2. Из папки проекта:

   ```bash
   npm start
   ```

   Одна команда поднимает **и базу, и оба сервера** — сайт (WEB) и серверную часть
   (API). В окне побегут строки с пометками `[API]` (жёлтые) и `[WEB]` (голубые).

3. Когда увидите строку `Local: http://localhost:5173/` — откройте в браузере
   **<http://localhost:5173>**.

**Окно терминала закрывать нельзя** — в нём работают серверы. Остановить всё —
`Ctrl + C`. База при этом продолжит работать в Docker; чтобы остановить и её:

```bash
npm run db:stop
```

Данные не пропадут.

### Проверка, что всё поднялось

Откройте <http://localhost:3000/api/health> — должно ответить
`{"status":"ok","db":"up"}`. Если нет — почти всегда **не запущен Docker Desktop**.

## 6. Открыть сайт на телефоне (для теста вёрстки)

Телефон и компьютер должны быть в **одной сети Wi-Fi**.

1. Запустите `npm start`.
2. В окне терминала среди строк `[WEB]` найдите адрес **Network** — например
   `http://192.168.1.42:5173/`. Это адрес для телефона.
   Если строки нет, узнайте IP компьютера:
   - macOS: `ipconfig getifaddr en0` (Wi-Fi) — покажет адрес вида `192.168.x.x`.
   - Windows: `ipconfig` → строка **IPv4-адрес** адаптера Wi-Fi.
3. На телефоне откройте браузер и введите `http://<этот IP>:5173`.

**Если не открывается:** первый раз система (брандмауэр Windows / firewall macOS)
может спросить разрешение для Node — разрешите для частной сети. Оба устройства
должны быть в одной сети без VPN. Код подтверждения при входе печатается
**в консоли на компьютере** (см. ниже), на телефон он не приходит.

## 8. Сброс базы (начать с чистого листа)

Стереть всех тестовых пользователей, заказы и вернуть заводское состояние.
Docker Desktop должен быть запущен:

```bash
npm run db:reset
```

Команда очищает базу, заново создаёт таблицы и заполняет стартовыми данными.
Спросит подтверждение — введите `y` и Enter. После сброса вход владельца снова
`Tuvafoto@mail.ru` / `TuvaFoto-2026`.

Совсем «с нуля» (снести и том Docker с данными):

```bash
docker compose down -v
npm run setup
```

## 9. Если что-то пошло не так

**`docker` ругается на подключение / сайт пишет `db:down`**
Не запущен Docker Desktop. macOS: `open -a Docker`; Windows — из «Пуска».
Дождитесь кита и снова `npm start`.

**«Port 5173 is already in use» или «Port 3000 is already in use»**
Серверы уже запущены в другом окне. Закройте то окно, либо остановите процессы:

- macOS: `lsof -ti:3000,5173 | xargs kill -9`
- Windows (PowerShell): `Get-Process -Name node | Stop-Process -Force`

**«command not found: npm / node»**
Node.js не установлен или терминал не перезапущен. Переустановите Node и откройте
новый терминал.

**Регистрация или вход выдаёт 502**
Не поднялась серверная часть. Запускайте именно `npm start` (а не `npm run dev` —
та поднимает только сайт), и следите, чтобы в окне были жёлтые строки `[API]`.

**Не приходит код подтверждения**
Он и не придёт письмом — почта не подключена. Код печатается в консоли среди
строк `[API]`.

**`Cannot find module '.prisma/client/default'` или `npm warn allow-scripts …` /
не собрался `argon2`, движки Prisma**

Две частые проблемы на **новых версиях Node (26+)**: не сгенерирован Prisma Client
и заблокированы install-скрипты пакетов — из-за чего не собрался нативный
`argon2` (им сид хэширует пароль владельца) и движки Prisma. Есть два пути.

*Вариант A (рекомендуется) — перейти на Node 22 LTS.* Снимает и блокировку
скриптов, и проблемы сборки нативных модулей под свежую Node:

```bash
brew install node@22
brew unlink node && brew link --overwrite --force node@22
node -v                                       # должно быть v22.x

rm -rf node_modules                            # чистая переустановка
npm install
npm run prisma:generate --workspace apps/api
npm run seed
npm start
```

*Вариант B — остаться на Node 26 и разрешить install-скрипты:*

```bash
npm approve-scripts --allow-scripts-pending
# если попросит выбрать — одобрите: argon2, esbuild, fsevents, prisma, @prisma/engines
npm install
npm run prisma:generate --workspace apps/api
npm run seed
npm start
```

Ключевой шаг в обоих — **`npm run prisma:generate --workspace apps/api`**: он
создаёт `.prisma/client`, которого не хватало. Начиная с текущей версии проекта
генерация уже входит в `npm run setup`, поэтому после `git pull` достаточно
одного `npm run setup`.

**macOS: «Docker Desktop» не запускается из `open -a Docker`**
Откройте Docker Desktop вручную из папки «Программы» (Applications) один раз и
подтвердите разрешения — дальше `open -a Docker` будет работать.

**Изменения не видны в браузере** — обновите страницу с очисткой кэша
(`Cmd/Ctrl + Shift + R`).

## 10. Шпаргалка команд

Все команды — из корневой папки проекта.

| Что нужно | Команда |
|---|---|
| Первичная подготовка (один раз) | `npm run setup` |
| **Запустить всё (база + серверы)** | `npm start` |
| Остановить серверы | `Ctrl + C` в окне терминала |
| Остановить базу | `npm run db:stop` |
| Сбросить данные в базе | `npm run db:reset` |
| Только сайт, без сервера (демо) | `npm run dev` |
| Проверить сервер | открыть <http://localhost:3000/api/health> |
| Запустить Docker (macOS) | `open -a Docker` |
